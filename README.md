"Introducción"

El presente informe describe el desarrollo, implementación y publicación de un programa en Python, cuyo objetivo es aplicar los conocimientos adquiridos sobre tipos de datos, identificadores y convenciones de nomenclatura. El trabajo fue desarrollado utilizando el entorno de desarrollo integrado PyCharm y posteriormente publicado en un repositorio público de GitHub, cumpliendo con las buenas prácticas de programación y organización del código.

El programa desarrollado está orientado a una aplicación básica en el ámbito de la arquitectura, específicamente al cálculo del área de una figura geométrica, lo cual permite relacionar la programación con situaciones reales del campo profesional.
Desarrollo

"3.1 Descripción del programa"

El programa desarrollado calcula el área de un rectángulo a partir de las dimensiones ingresadas por el usuario (largo y ancho). Posteriormente, valida si el área calculada es mayor que cero y muestra un mensaje indicando si el resultado es válido.

Esta funcionalidad es aplicable al ámbito de la arquitectura, ya que el cálculo de áreas es una actividad frecuente en el diseño y planificación de espacios.

"3.2 Tipos de datos utilizados"

Durante el desarrollo del programa se utilizaron los siguientes tipos de datos:

Float: Para representar medidas reales como el largo, el ancho y el área del rectángulo.

String: Para la entrada de datos mediante el teclado y la visualización de mensajes en pantalla.

Boolean: Para validar si el área calculada es mayor que cero.

Integer: Utilizado en comparaciones lógicas dentro de la validación.

El uso de estos tipos de datos garantiza un manejo adecuado de la información y resultados precisos.

"3.3 Identificadores y convenciones de nomenclatura"

Los identificadores utilizados en el programa siguen la convención snake_case, recomendada por el estándar PEP 8 de Python. Los nombres de variables son descriptivos y permiten comprender fácilmente su función dentro del programa.

"Ejemplos de identificadores utilizados:"

largo

ancho

area_rectangulo

area_valida

Esta práctica mejora la legibilidad, mantenimiento y comprensión del código.
